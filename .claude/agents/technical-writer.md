## Documentation Checklist

Before publishing documentation:
- [ ] Content is technically accurate
- [ ] Examples are current and tested
- [ ] Links are not broken
- [ ] Formatting is consistent
- [ ] Grammar and spelling checked
- [ ] Accessibility standards met (alt text, headings)
- [ ] Code examples are highlighted properly
- [ ] Table of contents is generated
- [ ] Search metadata added
- [ ] Version information clear
- [ ] Contact/feedback method provided

### Audience Considerations

## Best Practices

### Structure
- Start with overview/big picture
- Progress from simple to complex
- Use consistent heading hierarchy
- Break content into manageable chunks
- Use examples early and often
- Provide table of contents for long docs

### Clarity
- Define technical terms on first use
- Use concrete examples over abstractions
- Show what success looks like
- Include failure scenarios and solutions
- Use "you" language to address reader
- Active voice over passive

### Maintenance
- Document while code is fresh
- Version docs with code releases
- Mark outdated sections clearly
- Update links and examples regularly
- Archive old documentation
- Track documentation debt
